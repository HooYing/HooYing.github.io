2. Github Pages

GitHub Pages 是 GitHub 提供的一项服务，允许用户直接在 GitHub 仓库中托管静态网站。

网址：<username>.github.io

2.1 使用前提

  1. 仓库得是公开状态才能免费使用

  2. 仓库名称要命名为：<username>.github.io

2.2 使用流程

  1. 仓库Setting->Pages 选择Github Actions，然后配置工作流文件

    name: 部署文档

on:
  # 当代码push到branches分支时自动触发
  push:
    branches:
      # 确保这是你正在使用的分支名称
      - master

jobs:
  deploy-gh-pages:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3
        with:
          fetch-depth: 0
          # 如果你文档需要 Git 子模块，取消注释下一行
          # submodules: true

      - name: 设置 Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 22
       
      - name: 安装依赖
        run: npm install

      - name: 构建文档
        env:
          # 将nodejs内存限制设置为4GB 防止大型项目构建时内存不足
          NODE_OPTIONS: --max_old_space_size=4096
        run: npm run docs:build

      # - name: 部署到个人服务器
      #   uses: easingthemes/ssh-deploy@v2.0.7
      #   env:
      #     SSH_PRIVATE_KEY: ${{ secrets.ACCESS_TOKEN }}
      #     ARGS: "-avz --delete"
      #     SOURCE: "docs/.vuepress/dist"
      #     REMOTE_HOST: ${{ secrets.HOST }}
      #     REMOTE_USER: ${{ secrets.USER }}
      #     TARGET: ${{ secrets.TARGET }}
      
      - name: 部署到 GitHub Pages
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          # 网站根目录
          folder: docs/.vuepress/dist
          # 使用的仓库 不设置的话默认本仓库
          repository-name: HooYing/HooYing.github.io
          # 分支
          branch: main
          # 如果使用其它仓库 需要设置token
          token: ${{ secrets.DEPLOY_PRIVATE_KEY }}

  2. 生成token

    1）个人Settings->Developer Settings->Personal access tokens->Tokens

    2）创建token，一般给予repo权限就行

    3）将生成的token在仓库的流文件中用起来

    创建一个token的key-value：仓库Settings->Secrets and variables-> Actions->New respository secret

3. vuepress相关配置

3.1 主题配置

plugins.blog.excerptLength

  • 类型: number

  • 默认值: 300

    自动生成的摘要的长度。 注: 摘要的长度会尽可能的接近这个值。如果设置为 0，意味着不自动生成摘要。

markdown.revealjs

  • 类型: bool

  • 控制 @vuepress/plugin-revealjs，提供幻灯片支持。你可以设置 true 来直接启用它，或提供插件选项。

markdown.tasklist

  • 类型：bool
  • 启动任务列表：- [x] 一些文字

Catalog

  • 控制 @vuepress/plugin-catalog，提供目录的自动生成

  • 详情：

​ 该插件默认会全局注册一个 <Catalog /> 组件（除非你设置了 component 选项）。

​ 你可以在主题布局中或直接在 Markdown 文件中使用 <Catalog />。

​ 组件支持四个属性：

  • level：更改显示层次深度（最大仅支持 3 层），默认为 3。
  • base：显示指定文件夹的目录，默认显示当前文件夹目录。
  • index：为目录项添加索引号，默认无标号。
  • hideHeading：隐藏组件标题，默认会显示 目录 标题

3.2 frontmatter配置

article

  • 类型: boolean

  • 默认值: true

    是否将该文章添加至文章列表中
