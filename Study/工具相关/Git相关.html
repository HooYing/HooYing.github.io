1. 密钥相关

1.1 生成密钥

步骤如下：

  1. 先安装git
  2. 使用ssh-keygen命令生成密钥，其中-t表示类型是rsa类型（非对称加密），-C为邮箱地址
ssh-keygen -t rsa -C "xxx@qq.com"

执行完之后就会在~/.ssh目录下（windows在用户目录下C:\Users\{username}\.ssh\）生成两个文件：

  • id_rsa：私钥
  • id_rsa.pub：对应公钥

  1. 在仓库上（github、gitlab...）新增SSH Key，将公钥设置成id_rsa.pub文件里的内容。

  2. 设置用户名和邮箱

git config --global user.name "用户名"
git config --global user.email "邮箱"

1.2 一台电脑配置两个Git账号

  1. 生成新的密钥文件

    ssh-keygen -t rsa -f ~/.ssh/id_rsa_github -C "xxx@qq.com"

  2. 修改config文件

    ssh认证时，会自动尝试加载.ssh目录下的默认密钥文件（id_rsa）

    因此如果添加新的密钥文件，需要在.ssh/config文件中配置git服务域名和用户名使用哪个密钥文件

    # gitlab
Host gitlab.com
  HostName gitlab.com
  PreferredAuthentications publickey
  IdentityFile ~/.ssh/id_rsa
  User zhangsan # gitlab用户名
 
# github
Host github.com
  HostName github.com
  PreferredAuthentications publickey
  IdentityFile ~/.ssh/id_rsa_github
  User zhangsan123 # github 用户名

  3. 设置指定仓库使用对应账号来进行git提交

    在对应仓库下执行如下命令，指定对应的git账号来提交：

    git config user.email "xxx@xx.com"
git config user.name "zhangsan123" # 需要跟 config 文件里设置的用户名保持一致

2.2 Git本地操作相关命令

# 初始化git 
git init
# 暂存区
git add ....
# 查看代码状态
git status
# 提交
git commit -m "xxxxx"
# 查看提交的历史
git log
# 查看本地分支
git branch
# 查看所有分支，包括远程
git branch -a
# 创建分支，相当于在所在分支处设个快照出去
git branch name
# 删除分支
git branch -d name
# 切换分支
git checkout name
# 比较不同
git diff xxx xxx filename
# 回退commit
git reset --mixed|--soft|--hard commidid|HEAD^
# --mixed: 默认参数，回退到某个版本，文件还未add
# --soft: 回退到某个版本，文件已经add
# --hard: 回退到某个版本，文件的修改全部丢失

# 合并分支
# 1.先切换到要合并的分支
git checkout master
# 2. 将test分支合并到master
git merge test
# 3. 如果出现冲突，会提示合并失败，要先解决冲突
# 4. 如果不想合并了
git merge --abort
# 5. 如果冲突解决了直接add并commit就行了

# 合并某个分支下的某个文件或文件夹
git checkout master test.txt|dir/**

# 还原
git checkout -- aaa.html // 指定还原`aaa.html`文件
git checkout -- * // 还原所有文件

# 查看当前分支是从哪个分支创建来的：
git reflog --date=local | grep 分支名

# 合并其它分支的某次提交
git cherry-pick cid

# 合并多次commit
git rebase -i hash （前开后闭）
将后面的pick改为s
修改commit信息

# 临时保存未提交的修改, 方便切换其它分支
git stash       # 临时保存修改
git checkout other-branch
# 处理其他任务后返回
git checkout original-branch
git stash pop   # 恢复修改

# 设置代理 全局设置
git config --global http.proxy 'http://proxy.server:port'
git config --global https.proxy 'https://proxy.server:port'
# 查看设置情况
git config --global http.proxy
git config --global https.proxy

2.3 Git远程操作相关命令

# git远程
# 生成密钥
ssh-keygen -t rsa
# 测试是否连接成功
ssh -T https://github.com/HooYing/testGIt

# 设置用户名邮箱，这样提交之后commit就有对应的名称邮箱
# --global设置为全局
git config --global user.name "xxxx"
git config --global user.email "xxxx"
# 查看当前用户名和邮箱
git config user.name
git config user.email

# 关联仓库
git remote add origin git@github.com:HooYing/testGIt.git
本地新建的一个对应远程的仓库的名：origin

# 克隆仓库
git clone -b 分支名 git@github.com:HooYing/testGIt.git

# 查看有哪些远程仓库
git remote -v

# 将本地分支代码上传到远程分支，如果远程分支不存在，会创建
git push origin test

# 本地分支对应远程分支,跟踪远程分支
git branch --set-upstream-to=origin/master

# 推送当前分支并设置本地分支对应远程分支
git push --set-upstream origin ssl_support

# 切记在push代码时要先pull，如果不先pull，然后push前别人有新的提交，
# 那么会提示出错，并且自动会调用merge，将最新远程仓库的代码合并到当前本地的代码
git pull

# 远程仓库回退到某个版本
# 先本地回退
git reset --hard commitid|HEAD^
# 再强制覆盖提交到远程
git push --force origin master

# 远程分支强制覆盖本地
git fetch -all
git reset --hard origin/main
git pull --force origin main:main

# git 查看某个分支是从哪个分支拉出来的
# 先切换到需要查看的分支
git checkout 分支名
git reflog --date=local | grep 分支名

# 拉取大文件
git lfs pull

# 强制拉取远程覆盖本地
git fetch --all
git reset --hard origin/master

3. windows下利用Git实现将批量文件从LF转换为CRLF

从github直接下载的zip源码包，文件通常是LF格式的，如果想在windows上使用，需要转换为CRLF格式

这个时候就可以使用Git将批量文件从LF格式转换为CRLF：

# 初始化git
git init

# 设置core.autocrlf为true 一般windows下core.autocrlf默认为true
# core.autocrlf为true的作用：提交时自动将 CRLF 转换为 LF，检出时再将 LF 转回 CRLF，保持本地文件兼容性。
# 查看core.autocrlf的值
git config core.autocrlf
# 设置core.autocrlf为true
git config core.autocrlf true

# 随便弄个提交，保证后续git stash可使用
git add readme.txt
git commit -m "init"

# 暂存所有文件
git add .
git stash

# 恢复暂存文件（此时会自动应用CRLF转换）
git stash pop
